# Contributing

Thanks for your interest in improving HexMind.

This public repository is the open-core edition, so the best contributions are the ones that improve the engine, API, web UI, docs, tests, or sample assets without depending on the private commercial prompt library.

## Good First Contribution Areas

- Fix a reproducible bug in the CLI, API, or web UI
- Improve onboarding docs or examples
- Add tests around existing behavior
- Polish the sample persona and prompt assets
- Improve developer ergonomics in scripts and tooling

## Local Setup

### Backend

```bash
pip install -e ".[dev]"
pytest -q
```

### Frontend

```bash
cd web
npm install
npm run build
```

## Public Asset Boundary

Please avoid proposing changes that require the private hosted prompt library to understand or validate.

Public contributions should target:

- `personas/`
- `prompts/library/`
- `src/hexmind/`
- `web/`
- `tests/`
- `docs/public/`

## Pull Requests

- Keep pull requests focused
- Include tests when behavior changes
- Update docs when commands, public assets, or setup instructions change
- Prefer small, reviewable commits over large unrelated bundles

## Regenerating README Assets

The animated README asset and social preview images are generated by:

```bash
python scripts/render_readme_demo_gif.py
```

That script writes assets to `docs/public/assets/`.
